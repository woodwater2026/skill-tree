# Taxonomy Guide

Skill Tree uses a practical, scalable taxonomy.

## Rules

- Every skill gets **one** primary category.
- Secondary categories and tags are optional.
- Prefer the category that best describes the skill's main user outcome.

## Top-Level Categories

- Development
- Research
- Productivity
- Communication
- Data & Analysis
- Security & Compliance
- Agent Operations
- System / Environment
- Creativity

## Classification Heuristics

### Development
Use for skills whose main value is changing, generating, testing, or reviewing code.

### Research
Use for skills centered on information gathering, synthesis, or market/competitive analysis.

### Productivity
Use for organizational workflows, planning, notes, and internal task support.

### Communication
Use for messaging, email, posting, meeting support, or translation.

### Data & Analysis
Use for extraction, reporting, monitoring, visualization, and data processing.

### Security & Compliance
Use for auditing, permissions review, secret detection, and security/risk workflows.

### Agent Operations
Use for orchestration, memory, context, budget, routing, recovery, and agent runtime support.

### System / Environment
Use for local environment setup, file handling, dependency installation, browser/system automation.

### Creativity
Use for writing, design, ideation, media, or brainstorming-oriented skills.
